# Code of Conduct

This project follows a simple standard:

- be respectful
- be specific
- assume good faith
- critique code and ideas, not people
- keep security reports private until coordinated

We want this repo to feel safe for contributors, reviewers, and domain experts
who are joining the work from different backgrounds.

## Expected Behavior

- give actionable feedback
- explain tradeoffs honestly
- welcome questions from people who are new to the stack
- keep discussions technical, professional, and non-hostile

## Unacceptable Behavior

- harassment, intimidation, or personal attacks
- doxxing or sharing private information without consent
- deliberately misleading contributors about repo or deployment truth
- public disclosure of vulnerabilities before coordination

## Reporting

For conduct issues, contact the maintainer privately through the same GitHub
account used for this repository.

For security issues, use [SECURITY.md](SECURITY.md) instead.
